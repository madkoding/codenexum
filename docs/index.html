The problem

When your agent explores a repo, it gets buried under noise.

800
lines in a typical test log
4,000+
lines from a noisy grep
~70%
of those tokens are useless

CodeNexum intercepts that noise at the tool boundary and replaces it with indexed chunks, semantic summaries, and structured snippets. The model gets the signal, not the noise.

What you get

Every feature is enabled by default. Nothing to configure to start saving tokens.

🔥

FTS5 + BM25 index

Local SQLite, sub-millisecond search. Your code becomes searchable, not just readable.

🧠

Semantic summaries

800-line test log → 1 line: 8 passed, 3 failed. The model learns what it needs.

✂️

Smart compression

Strip ANSI colors, collapse repeated lines, trim stack indent. The details that cost tokens and add nothing.

🧩

Snippet substitution

read foo.ts returns the indexed symbols (functions, classes) instead of the whole file.

🔍

Indexed grep/glob

Search the real code, not your noisy filesystem. Loop-detection stops repeat reads.

🔗

Callers & impact

Ask "who uses this function?" and get the list instantly, no manual navigation.

💾

Smart cache

In-memory + on-disk. Repeated reads cost nothing. Persists across sessions.

📊

Live dashboard

Tokens saved, top queries, hot files, index health. See exactly where the savings come from.

🪶

Thin plugin

~120 LOC. All indexing logic lives in the app — the plugin is just an MCP proxy.

How it works

The interception loop, in three steps.

sequenceDiagram
    participant U as User
    participant OC as opencode
    participant P as @codenexum/plugin
    participant APP as CodeNexum
    participant DB as SQLite
    U->>OC: "find all callers of auth()"
    OC->>P: tool.execute.before
    P->>P: detect candidate
    OC->>OC: native read
    OC-->>P: tool.execute.after
    P->>APP: cm_read_snippet
    APP->>DB: SELECT chunks
    DB-->>APP: indexed symbols
    APP-->>P: compact snippet
    P-->>OC: substitute (smaller)
    P->>APP: log savings

Get started in 3 steps

Install the app

Open the downloaded file and launch CodeNexum. It runs in the background with a tray icon.

Add the plugin to opencode

In your opencode.json:

{
  "plugins": ["@codenexum/plugin"]
}

Restart opencode. The plugin auto-discovers the app.

Watch the savings roll in

Open the CodeNexum window to see live metrics: tokens saved, top queries, hot files, and the index health of every registered project.

When is it worth it?

Rule of thumb: break-even at 3–4 tool calls. Everything beyond is net gain.

✅

Long sessions, 20+ tool calls

Massive savings. The compression alone pays for itself many times over.

✅

Mid-size project exploration

First-time agent exploration of an unfamiliar repo — biggest win.

✅

Debugging test/build output

Semantic compression turns 800-line test logs into 1-line summaries.

❌

One quick question, no tools

Plugin overhead > savings. Skip it.

❌

Tiny projects (<20 files)

Nothing to index. Skip it.
