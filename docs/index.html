01

The problem

Agents get buried under noise. The signal is in there, somewhere.

When your agent explores a repo, every read, grep, and bash call returns a wall of text. A typical test log is 800 lines. A noisy grep can hit 4,000+. The model pays for every token — and most of those tokens are noise that never gets read.

CodeNexum sits at the tool boundary. It sees what the model is about to read, checks if there's a better representation in the local index, and substitutes a structured snippet when it's smaller. The model gets the signal. You stop paying for the noise.

What you get

Every feature ships enabled. No config to start saving.

FTS5 + BM25 index

Local SQLite, sub-millisecond search. Your code becomes searchable, not just readable.

Semantic summaries

800-line test log collapses to 8 passed, 3 failed. The model learns what it needs.

Smart compression

Strip ANSI, collapse repeated lines, trim stack indent. The details that cost tokens and add nothing.

Snippet substitution

read foo.ts returns indexed symbols (functions, classes) instead of the whole file.

Indexed grep/glob

Search the real code, not your filesystem. Loop-detection stops repeat reads cold.

Callers and impact

Ask "who uses this?" and get the list instantly. No manual navigation through 40 files.

Smart cache

In-memory + on-disk. Repeated reads cost nothing. Survives across sessions.

03

How it works

A 120-line plugin on top of a local app. The interception loop is three steps. 
sequenceDiagram
    participant U as User
    participant OC as opencode
    participant P as @codenexum/plugin
    participant APP as CodeNexum
    participant DB as SQLite
    U->>OC: "find all callers of authenticate()"
    OC->>P: tool.execute.before
    P->>P: detect candidate
    OC->>OC: native read
    OC-->>P: tool.execute.after
    P->>APP: cm_read_snippet
    APP->>DB: SELECT chunks
    DB-->>APP: indexed symbols
    APP-->>P: compact snippet
    P-->>OC: substitute (smaller)
    P->>APP: log savings

05

Get started

Three steps. Two minutes.
01

Install the app

Open the downloaded file and launch CodeNexum. It runs in the background with a tray icon — no window pops up unless you click it.

02

Add the plugin to opencode

Drop the plugin into your opencode.json:

{
  "plugins": ["@codenexum/plugin"]
}

Restart opencode. The plugin auto-discovers the app on the local network.

03

Watch the savings

Open the CodeNexum window to see live metrics: tokens saved per session, top queries, hot files, and the index health of every registered project.

06

When is it worth it?

Rule of thumb: break-even at 3-4 tool calls. Everything beyond is net gain.

Yes

  • Long sessions, 20+ tool callsThe compression alone pays for itself many times over.
  • Mid-size project explorationFirst-time agent exploration of an unfamiliar repo — biggest single win.
  • Debugging test/build outputSemantic compression turns 800-line logs into 1-line summaries.

Skip

  • One quick question, no toolsPlugin overhead exceeds savings.
  • Tiny projects (<20 files)Nothing meaningful to index.
